## 1. 목표

추세 정보원의 두 규칙(ma200·tsmom12)을 동일 엔진·net 기준으로 단독 백테스트하고,
buy&hold와 동일노출 정적 배분(equal_exposure) 두 대조군과 비교한다.

**DEFINITIONS 1.3 이진식 그대로 사용 (연속 스케일 변형 없음):**
- `ma200`:   `w = w_max if P_t > MA200_t else w_floor`
- `tsmom12`: `w = w_max if (P_t / P_{t-252} − 1) > 0 else w_floor`

## 2. 구현 파일

### `config/indicators/trend.yaml`

```yaml
rule:    "ma200"
w_floor: 0.0
# w_max는 base.yaml에서 상속 (1.0)
```

`w_floor=0.5` 변형 및 추세 강도 연속 스케일 변형은 추가하지 않았음 (M4 robustness 전용).

---

### `src/indicators/trend.py`

**신호 흐름 (ma200):**
`prices` → `rolling(200).mean()` → `P_t > MA200_t` 이진 분기 → `w ∈ {0, w_max}`

**신호 흐름 (tsmom12):**
`prices` → `P[i] / P[i-252] − 1` iloc 직접 참조 → `r12 > 0` 이진 분기 → `w ∈ {0, w_max}`

**`_assert_ma200_alignment(prices, ma200)`**
- `rolling(200, closed='right')` → 동일 인덱스 보장 → t 위치에서 P[t-199:t+1] causal ✓
- 인덱스 불일치·마지막 유효일 불일치 시 ValueError("룩어헤드")

**`_assert_tsmom_alignment(prices, w)`**
- 인덱스 동일성 단언
- 워밍업 경계: `w.iloc[:252]` 모두 NaN, `w.iloc[252]` non-NaN
- 위반 시 ValueError("룩어헤드")

**eval_start 규약:**
`standalone_data(data_raw, "trend")` → 1988-01-01 기준 입력 (sp500tr 첫날 1988-01-04로 inner join)
- ma200: rolling(200) 워밍업 후 eval_start = **1988-10-14**
- tsmom12: iloc lag=252 워밍업 후 eval_start = **1988-12-30**

**외부 데이터 없음:** sp500tr 가격에서만 계산. NFCI류 일절 사용 안 함.

### MA200 신호
- **휩소 특성**: 이진이라 방향성 모호한 구간에서 1990(18회)·2000·2007·2022(각 15회) 처럼 연 10회 이상 전환 → 비용 누적.
- **조용한 강세장 2013-2019**: avg_w=0.901, 전환 28회 — 대부분 w=1.0 유지. **Sharpe 0.633 vs ee 0.536** — ee 대비 명확한 우위 달성.

- **2022**: avg_w=0.187, 15회. 금리충격 조기 방어 + 반등 구간 휩소.
