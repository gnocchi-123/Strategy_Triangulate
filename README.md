# 다지표 위험 타게팅 연구 (Multi-Source Risk Targeting)

> **연구 목적:** 변동성·신용·추세 세 정보원으로 주식 노출을 조절하여 위험조정 성과 개선이 실재하는지 정직하게 검증.

## 데이터 출처 및 기간

| 시리즈 | 소스 | 가용 기간 | 역할 |
|---|---|---|---|
| S&P 500 총수익 (`^SP500TR`) | yfinance | 1988~ | 벤치마크 |
| VIX (`^VIX`) | yfinance | 1990~ | 변동성 대표 |
| VIX3M (`^VIX3M`) | yfinance | 2006~ | 변동성 기간구조 (보조) |
| 3M T-bill (`DTB3`) | FRED | 1988~ | 무위험수익률 |
| NFCI | FRED | 1971~ | **신용/금융여건 대표** |
| STLFSI4 | FRED | 1993~ | 신용/금융여건 변형·보조 |
| HY OAS (`BAMLH0A0HYM2`) | FRED | 2023-05-30~ | 신용 보조 (최근 robustness 전용) |

## 한계 및 주의사항

### hy_oas (BAMLH0A0HYM2) 데이터 단축 — 신용 대표 신호 재지정

**사실:** FRED는 2026년 4월부터 ICE BofA 계열(`BAML*`) 전체에 rolling 3년 윈도우 정책을 적용했다.
현재 `BAMLH0A0HYM2`는 2023-05-30~만 무료 제공되며, 이는 영구·진행성 제약이다.

**전 경로 확인 결과 (2026-06):**
- `fredgraph.csv` (키 없이): 2023-05-30~ (rolling 3년)
- FRED API (등록 키 보유): 동일
- ALFRED vintage `realtime_end=2026-01-01`: 2023-05-30~ (rolling 윈도우 내 데이터만 아카이브)
- ALFRED vintage `realtime_end=2023-01-01`: *"does not exist in ALFRED"* — 아카이브 자체 없음
- 결론: 1996~ 전체 이력은 모든 경로에서 획득 불가

**결정 (사전 등록 집합 변경 아님):**
`DEFINITIONS_AND_CONVENTIONS.md` 1.2절은 "HY OAS **또는** NFCI 등"을 신용 정보원으로 명시한다.
신용 신호의 가치는 위기 구간 방어력인데 2023-05-30~ 3년에는 신용 위기 표본이 없어
단독 검증이 성립하지 않는다. 짧은 표본의 "노출 축소 손해"는 신호 실패가 아닌 표본 실패로
결론이 오염된다.

따라서 **신용/금융여건 대표 신호를 NFCI(1971~)로 재지정**한다.
STLFSI4(1993~)는 변형·보조. hy_oas는 제거하지 않고 최근 구간 robustness·향후 재검증 전용으로 보존.
파라미터: `config/indicators/credit.yaml` 참고.

### VIX3M 짧은 표본

`^VIX3M`은 2006~. 기간구조 피처는 표본 제약 — 보조 변형으로만 사용.

### 표본 길이 비대칭

단독 평가는 각자 최대 기간, 정보원 간 비교·결합은 공통 기간. 이 비대칭은 사실이며 우열이 아니다.
