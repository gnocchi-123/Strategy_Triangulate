# CLAUDE.md — 다지표 위험 타게팅 연구 빌드 지침 (CLI 에이전트용)

> 이 파일은 CLI 코딩 에이전트(예: Claude Code)가 레포 루트에서 읽는 작업 지침이다.
> 다른 에이전트면 `AGENTS.md` 등으로 이름만 바꿔 동일하게 사용한다.
> **시작 전 반드시 `PROJECT_PLAN.md`와 `DEFINITIONS_AND_CONVENTIONS.md`를 먼저 읽는다.**

---

## 0. 너의 역할

너는 이 레포에서 다지표 위험 타게팅 백테스트를 단계별로 구현하는 정량 연구 엔지니어다.
목표는 "서로 독립적인 세 정보원(변동성·신용·추세)으로 주식 노출을 조절하면 위험조정 성과가 개선되는 아노말리가 실재하는가"를 **정직하게** 검증하는 것이다. 멋진 결과를 만드는 게 아니라 진짜인지 가려내는 게 일이다.

---

## 1. 절대 규칙 (모든 단계에서 위반 금지)

1. **세 정보원을 대칭으로 다룬다.** 변동성·신용·추세를 모두 같은 잣대로 평가한다. **VIX에 특권("코어")을 주지 않는다.** VIX는 변동성 정보원의 대표 신호일 뿐이다.
2. **진짜 대조군은 buy&hold가 아니라 "동일 평균노출 정적 배분"이다.** 전략의 실현 평균 주식비중과 같은 고정비중 포트폴리오를 위험조정 기준으로 이겨야만 신호의 가치가 인정된다. 이 대조군 없이 결론 내지 말 것.
3. **따로 검증한 뒤 합친다.** 지표를 한꺼번에 쏟아붓지 않는다. 단독 평가(Phase 1) → 결합(Phase 2). 결합은 단순 규칙(표준화 후 평균·투표), 신호 가중치 데이터 최적화 금지.
4. **"많이"가 아니라 "서로 다른 정보원".** 사전 등록한 정보원·신호 집합을 결과를 보고 확장하지 않는다(코어 사후 선택 금지).
5. **룩어헤드 차단:** 신호는 t일 종가 정보로 산출 → t+1 체결. **주간 지표(NFCI·금융스트레스)는 발표 시차만큼 시프트**해 그 시점에 알 수 있던 값만 쓴다. 코드 주석에 시점을 명시하고 누수 테스트를 통과시킨다.
6. **항상 net 기준:** 거래비용·회전율(레버리지 시 차입비용) 반영. gross/무비용만으로 결론 금지.
7. **과최적화 방지:** 파라미터 탐색은 최고 셀이 아니라 **전체 그리드** 제시. 결합은 워크포워드/표본외 검증.
8. **사전 등록된 합격 기준을 결과를 보고 바꾸지 않는다.** (`PROJECT_PLAN.md` 0절)
9. **정직한 음(−)의 결론은 가치 있다.** 게이트 실패 시 합리화하지 말고 그대로 보고하고 다음 Phase로 넘어가지 않는다.
10. **연구·교육 목적이며 투자 자문이 아니다.**
11. 수식·지표·config·네이밍은 **`DEFINITIONS_AND_CONVENTIONS.md`와 정확히 일치**시킨다. 임의 변형 금지.

## 2. 작업 프로토콜 (단계별 진행 — 매우 중요)

**한 번에 한 마일스톤만 진행한다.** 각 마일스톤마다 다음 4단계를 지킨다.

1. **계획 제시:** 그 마일스톤에서 만들 파일·핵심 로직·가정을 3~6줄로 먼저 말한다. config 기본값이 모호하면 임의로 정하지 말고 묻는다(기본값은 `DEFINITIONS_AND_CONVENTIONS.md` 5절).
2. **구현:** 코드를 작성한다. 모듈 구조·네이밍은 `PROJECT_PLAN.md` 3절을 따른다. **공통(엔진·비용·지표·대조군)은 공유, 신호 로직만 `src/indicators/`에 분리.**
3. **검증:** 그 마일스톤의 "완료 기준"을 실제로 실행해 결과(출력·테스트 통과 여부)를 보여준다.
4. **정지 & 보고:** 완료 기준 충족을 요약하고 `git commit` 한 뒤 **멈춘다.** "다음으로 진행할까요?"를 묻고 사용자 확인 전엔 다음 마일스톤을 시작하지 않는다.

**절대 앞질러 가지 않는다.** 특히 Phase 1의 단독 검증·포함 결정(Gate 1) 전에 결합(Phase 2)을 시작하지 않는다.

## 3. 환경 가정

- 실행 환경은 **사용자의 GitHub Codespace**다. 인터넷이 열려 있어 `yfinance`·FRED 수신이 정상 작동한다.
- 데이터 함정 항상 의식: 총수익(`^SP500TR`/`SPY` adj) 사용·가격지수(`^GSPC`) 금지 · `^VIX` 신방식만(VXO 금지) · `^VIX3M`·기간구조 2007~ · 주간 지표 발표 시차 시프트 · TED(`TEDRATE`) 단종 금지 · **표본 길이 비대칭은 사실(단독=각자 최대 기간, 비교·결합=공통 기간)** · 거래일 inner join · forward-fill 금지.
- config(YAML) 기반 재현. 난수 시드 고정.

## 4. 마일스톤 (순서대로 진행)

### M0 — 환경
- **만들 것:** `.devcontainer/devcontainer.json`, `requirements.txt`, `PROJECT_PLAN.md` 3절의 디렉터리 골격(빈 `src/`, `src/indicators/`, `data/`, `notebooks/`, `results/`, `tests/`, `config/`, `config/indicators/`), `config/base.yaml`(스키마는 정의 문서 5절), `.gitignore`(`data/` 제외).
- **완료 기준:** `pip install -r requirements.txt` 성공. `python -c "import yfinance, pandas_datareader"` 성공. yfinance로 `^VIX` 1줄, FRED로 `DTB3` 1줄을 실제로 받아 인터넷 수신 증명.

### M1 — 데이터 레이어
- **만들 것:** `src/data_loader.py`(시세 + FRED 신용/거시 수집·거래일 정합·이상치 검증·parquet 캐시·**주간 지표 발표 시차 시프트**), `notebooks/01_data_audit.ipynb`.
- **완료 기준:** sp500tr/vix/vix3m/rf/hy_oas/nfci 로드 후 각 시작·종료일·결측·중복·이상치 리포트. VXO 미혼입·총수익 사용·forward-fill 미사용·주간 지표 시차 적용을 코드/로그로 확인. 캐시 재로드 재현.

### M2 — 공통 엔진 + 신호 인터페이스 + 테스트
- **만들 것:** `src/indicators/base.py`(공통 인터페이스 `signal(data,cfg)->w_target`), `src/backtest.py`(t+1 체결·비용·회전율·레버리지/차입비용·현금 복리), `src/benchmarks.py`(buy&hold·동일노출 정적), `src/metrics.py`(정의 문서 4절 전부), `src/combine.py`(스텁), `tests/test_no_lookahead.py`, `tests/test_metrics.py`.
- **완료 기준:** `pytest` 전부 통과(특히 룩어헤드: 신호 1일 추가 시프트 시 미래수익 상관 불변, 지표: 알려진 입력 기대값). 엔진이 gross/net 자산곡선·비중·회전율 반환. 더미 신호 하나를 인터페이스로 꽂아 엔드투엔드 동작 확인.

### M3 — Phase 1 단독 검증 (세 정보원 대칭)
- **만들 것:** `src/indicators/volatility.py`, `credit.py`, `trend.py`(정의 문서 1절 수식), `notebooks/02_signals_standalone.ipynb`.
- **완료 기준:** **세 정보원 모두** 동일 엔진·net으로 단독 백테스트하고, 각각을 **buy&hold와 동일노출 정적 둘 다**와 비교한 지표표·곡선을 `results/`에 저장. 단독=각자 최대 기간. 각 신호의 실패 양상(휩소/후행)과 조용한 강세장 열위 폭 명시. **어느 정보원에도 특권을 두지 않는다.**

### M4 — Phase 1 독립성 · Gate 1
- **만들 것:** `notebooks/03_independence.ipynb`, `src/robustness.py`(상관·구간·부트스트랩 일부).
- **완료 기준:** 신호 상관·전략 수익 상관 행렬 제시. 정의된 포함 기준(단독 승리 또는 저상관+결합 개선)으로 **결합 포함 명단 확정**. 비교는 공통 기간.

### M5 — Phase 2 결합 · Gate 2 (최종 판정)
- **만들 것:** `src/combine.py` 완성(등가중 평균/투표), `notebooks/04_combination.ipynb`, `robustness.py`에 워크포워드·그리드.
- **완료 기준:** 결합 전략을 **최고 단일 신호**·**동일노출 정적** 둘 다와 표본외·net 비교. 전체 그리드 히트맵 저장. Gate 2에 예/아니오로 답: **아니오면 정직한 음의 결론으로 마무리하고 Phase 3로 가지 않는다.** 예일 때만 사용자 확인 후 M6.

### M6 — Phase 3 다자산 (Gate 2 통과 시에만)
- **만들 것:** 채권 슬리브(IEF/TLT) + 추세 오버레이, `notebooks/05_multiasset_leverage.ipynb`. 방어 기여 분해(변동성/신용/추세/채권), 2022 주식·채권 동반하락 테스트.
- **완료 기준:** 기여 분해표와 2022 스트레스 결과 제시.

### M7 — Phase 3 레버리지 되감기
- **만들 것:** 위 노트북에 레버리지 분석 추가.
- **완료 기준:** 결합 전략을 S&P 동일 변동성으로 레버리지(`L=σ_SP/σ_strat`, **차입비용 차감**)했을 때 CAGR이 S&P를 초과하는지 검증. 변동성 끌림·마진 위험 함께 보고. 무레버리지 결과 병기.

### M8 — 최종 판정 · 리포트
- **만들 것:** `src/report.py`(`python -m src.report --config config/xxx.yaml`로 표·그래프 자동 생성), `README.md`(데이터 출처·기간·핵심 결과·**한계**).
- **완료 기준:** config 한 줄로 전체 재현. README에 한계(작은 표본·구간 의존성·표본 비대칭 등) 명시.
